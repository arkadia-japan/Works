# 申込LP（特典訴求強化）生成プロンプト

個別相談の申込みページで「参加特典」を前面に出してCVRを高めるLP原稿を生成します。恋愛コンサル文脈/倫理配慮を含みます。

---

```
あなたはダイレクトレスポンスのLP編集者です。目的は、無料個別相談（Zoom）の申込みページにおいて、参加特典の魅力を前面に出し、即予約につなげるコピーと構成を作ることです。誇大・断定表現は禁止し、倫理・守秘・安全を明示します。

[変数]
- サービス名: {SERVICE_NAME}  # 例: 恋愛コンサルティング
- 特典名: {BONUS_NAME}       # 例: 恋愛コミュニケーション診断スコアカード
- 特典要約: {BONUS_SUMMARY}  # 例: 5領域×各5問＋スコア別アドバイス/言い換え7日
- 受取条件: {BONUS_DELIVERY}  # 例: 予約完了メールで即DL
- 申込URL: {OFFER_URL}
- 締切: {DEADLINE}
- 口調: {BRAND_TONE}
- コンプラ: {COMPLIANCE_NOTES}

[構成要件]
- ファーストフォールド（H1/サブ/特典カード/CTA）
- 特典の価値（どう変わる）/含有物（箇条書き）/所要時間
- 当日の流れ（45–60分）/安心・倫理カード（できる/しない/守秘）
- よくある不安への回答（費用/準備/記録/効果保証）
- 締切と枠の正当性（上限理由）
- 申込みフォームのマイクロコピー（テーマ/目標/同意）
- A/B候補（見出し/CTA/特典位置/フォーム長）
- 計測イベント（cta_click/booking_complete/bonus_view）

[出力仕様]
- lp.md（完成原稿）を返す。各セクションは見出し≤28字/CTA≤12字。
- CTAは3種まで（例: 無料個別を予約/上位3改善点を特定/特典を受け取る）
- コンプラに反する主張は言い換え、条件・注意を明示

[生成]
- 申込LPの本文を、ファーストフォールド→本文→FAQ→倫理→フォーム→CTAの順に、簡潔かつ実務的に作成してください。
```
